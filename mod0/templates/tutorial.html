What is MOD0?

MOD0 is an additional Python development environment that uses the repository-first architecture pattern, allowing you to work with a separate repository while keeping your TKT0 environment untouched.

Repository-First Architecture

The key principles of repository-first architecture include:

  • All application code comes from external repositories
  • Configuration is kept separate from application code
  • Multiple repositories can be used in the same environment
  • Easy switching between different projects

With MOD0, you can experiment with additional repositories without affecting your main TKT0 environment.

Adding a New Repository

To configure a repository for MOD0:

Step 1: Edit the .env file

# Edit the .env file
nano /home/turtle_wolfe/repos/devops-extract/.env

# Add a line for MOD0_REPO_URL
MOD0_REPO_URL=https://github.com/username/repository.git

Step 2: Start the MOD0 Environment

# Navigate to the project root
cd /home/turtle_wolfe/repos/devops-extract

# Start the MOD0 service
./start.sh mod0

Working with Multiple Environments

Benefits of using both TKT0 and MOD0:

  • Compare two different Flask projects side by side
  • Test migration of code from one repository to another
  • Maintain a stable environment while experimenting with new code
  • Run two Python applications with different dependencies

Accessing Your Environments

TKT0 is available at:

  • Flask: http://tkt0.localhost or http://localhost:5000
  • Jupyter: http://jupyter.localhost or http://localhost:8888

MOD0 is available at:

  • Flask: http://mod0.localhost or http://localhost:5001
  • Jupyter: http://mod0-jupyter.localhost or http://localhost:8889
